# 2025 中国研究生创"芯"大赛·EDA 精英挑战赛 —— 赛题三：版图多层链路追踪算法

> **命题单位**：杭州广立微电子股份有限公司

---

## 一、赛题背景

芯片版图链路追踪（Net Trace）技术是芯片设计与制造中的关键技术之一，目前已成为提升芯片良率、降低设计风险的重要手段。该技术在芯片设计和制造的应用场景包括：

1. **芯片设计阶段连接性验证与一致性检查**：Net trace 技术通过追踪版图中的物理连线，确保实际版图与逻辑原理图（如 SPICE 网表）的连接关系完全一致。例如，在 LVS（版图与原理图一致性检查）[1] 过程中，该技术可识别短路、断路或信号路径偏差，避免因设计错误导致的功能失效；
2. **制造缺陷定位与良率提升** [2]：通过追踪晶圆测试中的失效点，结合版图数据，Net trace 可快速定位工艺波动或光刻异常导致的物理缺陷（如金属桥接或孔洞），优化工艺参数；
3. **设计优化**：如寄生参数提取流程中通过 Net trace 技术追踪金属连线的几何形状和材料特性 [3]，提取寄生电阻、电容和电感（RC Extraction），用于电路仿真。在功耗与散热优化流程 [4] 中通过 Net trace 技术分析电源网络（Power Grid）的电流密度和压降（IR Drop），避免局部过热或供电不足。

在先进工艺节点下，随着芯片规模和复杂度的指数级增长，版图链路追踪技术面临数据量激增、多层连接性关系复杂度提升等多重挑战，推动着 EDA 工具和算法的持续创新。

版图链路追踪技术属于数字/模拟/射频版图的物理验证领域，主要应用在前述的芯片设计验证、优化和制造良率提升阶段。本赛题将模拟工业界真实的版图链路追踪场景，输入为待分析版图、版图层级连接性关系和追踪起点，输出为追踪全路径结果。

## 二、赛题描述

链路追踪的最基础应用为输入起点层、起点坐标和层之间的连接关系，在版图文件中搜索起点所在多边形的连通区域，本赛题的目标即为实现此功能。

### 2.1 输入文件描述

本赛题将提供 1 个版图文件和 3 个规则文件。

#### 规则文件

由**起点**、**Via 规则**和 **Gate 规则**组成。本赛题用文本文件来描述规则文件，其格式如图 1 所示。包含两部分：

- **第一部分**：描述规则文件中包含的起点信息。以 `StartPos` 开始。紧随的一行描述了起点的层（如 L2）及坐标，表示链路追踪的起点从 L2 层的某个位置开始。
- **第二部分**：描述 Via 规则信息，以 `Via` 开头。紧随其后描述了连接关系（如 L1、L2 和 L3 的 Via 关系），表示 L1 通过 L2 与 L3 相连——即 L1 与 L2 上彼此相交的多边形视为同一个连通区域，L2 与 L3 上彼此相交的多边形视为同一个连通区域。

> 图示：图 1a、图 1b（单/多个 Via 规则示例）

**请注意以下几点**：

1. 每个规则文件可以有**一个或两个起点**、**一个或多个 Via 规则**和**至多一个 Gate 规则**；
2. 起点的坐标值为 **32 位有符号整数**；
3. Via 规则中每个 Layer 仅包含**字母、数字和下划线**。每个 Via 规则至少 1 个 Layer，不含重复 Layer。包含多个 Layer 的 Via 规则中，**相邻的 Layer** 上彼此相交的多边形视为同一个连通区域，**不相邻的 Layer** 间无连接性关系。
   - 如图 1a 的单个 Via 规则：L1 与 L2 或 L2 与 L3 相交的图形视为相互连通，但 L1 与 L3 相交的不连通。
   - 如图 1b 的两个 Via 规则：L1 与 L3、L3 与 L4、L2 与 L3 层都存在连通关系，但 L1 与 L4、L2 与 L4、L1 与 L2 之间不能直接连通。重复定义的连接性关系将被忽略。
4. **Gate 规则**：指定芯片版图中的 **Poly（多晶硅）** 和 **AA（有源区）** 层，仅包含两层——第一层为 Poly，第二层为 AA。
   - 如果 Poly 被连接到高电平信号，与之相交的 AA 两侧区域（Source 和 Drain）处于连通状态；
   - 如果 Poly 被连接到低电平信号，与之相交的 AA 两侧区域处于断开状态。
   - 如图 2a，P1 接高电平、P2 接低电平，则 S1 和 D1 连通，S2 和 D2 断开。

**多边形相交条件**：以下三种情况均属于相交——
- 面积重叠（重叠区域面积 > 0）
- 边界点相交
- 边界线相交

> 图示：图 2a、图 2b

#### 版图文件

本赛题采用**文本文件**描述版图（非 GDSII/OASIS）。文件包含若干层的若干个**曼哈顿多边形**（所有边平行或垂直于坐标轴，不含内孔和自交）。

- 每层多边形以 `Layer` 标识开头，到下一个 `Layer` 标识或文件结尾截止
- Layer 标识之间的每一行按**逆时针方向**描述一个多边形各顶点坐标
- 坐标值为 **32 位有符号整数**
- Layer 标识只包含字母、数字和下划线
- 注解格式 `#Layer@#num` 表示该多边形为某层上第 num 个多边形（顺序与输入一致）

> 图示：图 3、图 4

### 2.2 链路追踪要求

1. 从规则文件中的起点开始，在版图文件中搜索图形，满足 Via 规则和 Gate 规则，输出与起点连通的所有多边形坐标。结果按 Layer 分组，顶点按逆时针顺序输出。
   - 示例：图 5 用 via 规则 `L1 L2 L3` 在图 3 版图中搜索，红色十字为 L3 层第 2 个多边形内的起点，红色粗线多边形为搜索结果。输出：L1@1, L1@2（L1 层）；L2@2（L2 层）；L3@1, L3@2（L3 层）。L3@3 因为 L3 与 L1 在 Via 规则中不相邻而不能到达。

> 图示：图 5、图 6

2. **三个子任务**：

   - **子任务一**：单个 Layer + 单个起点。Via rule 仅含一个 Layer，与起点 Layer 相同。从起点出发，正确连通结果应包含版图中所有多边形。（图 7）

   - **子任务二**：多个 Layer + 单个起点。需考虑 Via 规则，在不同 Layer 间寻找连通多边形。输出所有连通各层多边形结果。

   - **子任务三（Gate 规则）**：模拟芯片逻辑电路晶体管开关状态。
     1. 从第一个起点 **s1**（Poly 输入位置）出发，接入高电平信号后，与之有连接性关系（考虑多层 Via rule）的 Poly 全部置为高电平，其余 Poly 为低电平；
     2. 在此状态下，从第二个起点 **s2** 开始搜索，考虑多层 Via rule，输出所有与 s2 连通的各层多边形。**与 s1 连通的各层多边形不需要输出**。

   - 示例：图 9 规则 + 图 8 版图 → 图 10 结果。Gate rule：PO（Poly）、AA（AA 层）。Via rule：M1 ↔ CT ↔ PO。s1 在 M1 层 (118743, 438448)，左侧 PO 获高电平，上方 AA 左右连通；右侧 PO 低电平，AA 断开。红色边框多边形为最终输出。

   **注意**：
   - AA 层多边形 P1 与 Poly 层多边形 P2 相交时，P2 **必定贯穿** P1（将 P1 分成两部分）
   - 若 AA 多边形 P1 与 s2 连通且与 Poly 多边形 P2 相交，需计算 P1 被 P2 切割后的多边形再放入结果
   - 图 10 中上方 AA 被切割为 AA@1、AA@2、AA@3（贯穿 AA@1/AA@2 的 PO 与 s1 连通，AA@1 与 AA@2 连通）；下方 AA 被切割为 AA@4、AA@5、AA@6（贯穿 AA@5/AA@6 的 PO 不与 s1 连通，AA@5 不在结果中）

> 图示：图 7、图 8、图 9、图 10

### 2.3 输出文件格式

- Linux 系统文本文件格式
- 当前规则文件中起点在版图文件搜索到的所有结果合集
- 按照 Layer 对多边形分组，每个多边形占一行
- 多边形顶点按逆时针顺序输出

### 2.4 运行语法

```shell
trace -layout ./layout.txt -rule ./rule.txt [-thread n] -output ./res.txt
```

参数说明：

| 参数 | 必选 | 说明 |
|------|------|------|
| `-layout` | 是 | 版图文件路径 |
| `-rule` | 是 | 规则文件路径 |
| `-thread n` | 否 | 指定 n 个线程并行计算 |
| `-output` | 是 | 输出结果文件路径 |

2. 开发中所有调用的第三方库信息：库名及其官网
3. 鼓励提交源码

## 三、评分标准

得分点分为 3 部分：**正确性**、**性能**、**多线程并行计算**。

### 3.1 正确性（满分 60 分）

- 三个子任务，每个完全正确（无错误/遗漏多边形）得 20 分
- 每有 1 个错误或遗漏多边形，扣 1 分，扣完为止
- 三项得分相加为正确性总分

### 3.2 性能（满分 40 分）

> 仅在**正确性满分**时可得性能分

- **单线程性能分（20 分）**：按单进程单线程运行时间排名。第 1 名 20 分，之后每降一名扣 `20 / (参与排名人数 - 1)` 分。若未设定 `-thread` 却使用了多进程/多线程，记 0 分。
- **多线程性能分（20 分）**：按单进程 8 线程（环境保证 ≥8 CPU 核）运行时间排名。未实现多线程则用单线程时间。若最大线程数超过 `-thread` 设定值，记 0 分。

### 3.3 其他说明

若 Hidden case 单进程单线程运行时间**超过 8 小时**，正确性和性能得分均为 0。
